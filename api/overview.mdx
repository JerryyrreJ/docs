Sub exposes a v1 REST API for automating subscription management. Use it to sync data with other tools, build workflows, or manage subscriptions from scripts. The API follows REST conventions, accepts JSON request bodies, and returns JSON responses. Every request must be authenticated with an API key.

## Base URL

Your base URL depends on how you are running Sub. If you are using the hosted app, use `https://sub.jerrylu.app`. If you have deployed your own instance, replace this with your deployment URL.

```
https://your-site.example
```

All API endpoints are served under the following base path:

```
/api/v1/subscriptions
```

## Available Endpoints

| Method | Path | Description |
|---|---|---|
| `GET` | `/api/v1/subscriptions` | List all your subscriptions |
| `GET` | `/api/v1/subscriptions/:id` | Get a single subscription |
| `POST` | `/api/v1/subscriptions` | Create a subscription |
| `PATCH` | `/api/v1/subscriptions/:id` | Update a subscription |
| `DELETE` | `/api/v1/subscriptions/:id` | Delete a subscription |

## Authentication

Every request must include your API key as a Bearer token in the `Authorization` header. API keys are prefixed with `subm_` and are tied to your Sub account.

```bash
Authorization: Bearer subm_YOUR_KEY
```

You generate and manage API keys from the **Developer API** section of the user menu. See the [Authentication](/api/authentication) page for full details.

<Note>
  The API requires a Sub account with cloud sync enabled and Netlify Functions running on your deployment. If you are running Sub in local-only mode without cloud sync configured, the API endpoints are not available.
</Note>
