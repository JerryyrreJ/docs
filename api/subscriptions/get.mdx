The Get endpoint retrieves a single subscription by its unique ID. Use this when you already know the subscription's UUID and want to fetch its current state without loading your entire list. The server returns `404` if the subscription does not exist or belongs to a different account.

## Endpoint

```
GET /api/v1/subscriptions/:id
```

## Path parameters

<ParamField path="id" type="string" required>
  UUID of the subscription to retrieve. Must be a valid UUID v4, for example `a1b2c3d4-e5f6-7890-abcd-ef1234567890`.
</ParamField>

## Request example

```bash
curl -H "Authorization: Bearer subm_YOUR_KEY" \
  https://sub.jerrylu.app/api/v1/subscriptions/a1b2c3d4-e5f6-...
```

## Response

Returns `200 OK` with an envelope containing the matching subscription object under `data`.

<ResponseField name="data" type="object" required>
  The subscription object for the requested ID.

  <Expandable title="Subscription object fields">
    <ResponseField name="id" type="string" required>
      Unique UUID assigned by the server when the subscription was created.
    </ResponseField>

    <ResponseField name="name" type="string" required>
      Human-readable service name, for example `Netflix`.
    </ResponseField>

    <ResponseField name="category" type="string" required>
      Category label assigned to this subscription.
    </ResponseField>

    <ResponseField name="amount" type="number" required>
      Cost per billing period expressed as a decimal number.
    </ResponseField>

    <ResponseField name="currency" type="string" required>
      ISO 4217 currency code. One of: `CNY`, `USD`, `EUR`, `JPY`, `GBP`, `AUD`, `CAD`, `CHF`, `HKD`, `SGD`.
    </ResponseField>

    <ResponseField name="period" type="string" required>
      Billing frequency. One of: `monthly`, `yearly`, `custom`.
    </ResponseField>

    <ResponseField name="lastPaymentDate" type="string" required>
      Date of the most recent payment in `YYYY-MM-DD` format.
    </ResponseField>

    <ResponseField name="nextPaymentDate" type="string" required>
      Server-calculated date of the next payment in `YYYY-MM-DD` format.
    </ResponseField>

    <ResponseField name="customDate" type="string">
      Custom billing interval used when `period` is `custom`. The number of days between payments, expressed as a string integer (for example `"45"`). Omitted from the response when `period` is `monthly` or `yearly`.
    </ResponseField>

    <ResponseField name="notificationEnabled" type="boolean" required>
      Whether payment-reminder notifications are enabled for this subscription.
    </ResponseField>

    <ResponseField name="createdAt" type="string" required>
      ISO 8601 timestamp of when the subscription was created.
    </ResponseField>

    <ResponseField name="updatedAt" type="string" required>
      ISO 8601 timestamp of the most recent update.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="requestId" type="string" required>
  Unique identifier for this API request. Include this value when contacting support.
</ResponseField>

## Errors

| HTTP status | Error code | Description |
|---|---|---|
| `400` | `invalid_subscription_id` | The `:id` path segment is not a valid UUID. |
| `404` | `subscription_not_found` | No subscription with that ID exists for your account. |

## Rate limit headers

Every response includes the following headers so you can track your usage.

| Header | Description |
|---|---|
| `X-RateLimit-Limit` | Maximum requests allowed in the current window. |
| `X-RateLimit-Remaining` | Requests remaining before you are rate-limited. |
| `X-RateLimit-Reset` | Unix timestamp when the current window resets. |
