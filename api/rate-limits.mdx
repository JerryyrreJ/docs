Sub enforces rate limits on API requests per user per hour to ensure fair usage across all accounts. Every API response includes headers that tell you your current quota, how many requests you have left, and when your window resets — so you can pace your requests accordingly.

## Rate Limit Tiers

Your hourly request quota depends on your plan. The limit applies to all API keys on your account combined, not per individual key.

| Plan | Requests per Hour |
|---|---|
| Free | 60 |
| Premium | 1,000 |

## Rate Limit Response Headers

Every API response — whether successful or not — includes the following headers.

| Header | Description |
|---|---|
| `X-RateLimit-Limit` | Your total quota for the current one-hour window |
| `X-RateLimit-Remaining` | Number of requests you can still make before the window resets |
| `X-RateLimit-Reset` | Unix timestamp (seconds) indicating when your quota window resets |

Here is an example of what these headers look like on a response:

```
X-RateLimit-Limit: 60
X-RateLimit-Remaining: 47
X-RateLimit-Reset: 1717200000
```

## Handling a Rate Limit Error

When you exceed your quota, the API returns HTTP `429` with the following response body. The rate limit headers are still present on this response so you can determine when to retry.

```json
{
  "error": {
    "code": "rate_limit_exceeded",
    "message": "Rate limit exceeded"
  },
  "requestId": "a1b2c3d4-e5f6-7890-abcd-ef1234567890"
}
```

To recover, wait until the Unix timestamp in `X-RateLimit-Reset` has passed, then resume sending requests. You do not need to take any other action — your quota resets automatically at the start of each hour.

<Tip>
  Check `X-RateLimit-Remaining` before sending a batch of requests. If the value is lower than the number of requests you plan to make, pause until `X-RateLimit-Reset` before starting the batch. This avoids hitting the limit partway through and leaving your data in a partially updated state.
</Tip>

<Tip>
  If 60 requests per hour is not enough for your use case, upgrade to Premium to raise your quota to 1,000 requests per hour. You can upgrade from the [Premium page](/premium/upgrade).
</Tip>
