sidebarTitle: "Authentication"

---

Every Sub API request must include a valid API key in the `Authorization` header. Keys are prefixed with `subm_` and tied to your account. Without a valid key, the API rejects your request with a `401` status and an `invalid_api_key` error code.

## Creating an API Key

<Steps>
  <Step title="Sign in to Sub">
    Open Sub and sign in to your account.
  </Step>
  <Step title="Open the Developer API menu">
    Click your user avatar or name to open the user menu, then select **Developer API**.
  </Step>
  <Step title="Click Create key">
    Press the **Create key** button to start generating a new API key.
  </Step>
  <Step title="Name your key">
    Give the key a descriptive name (up to 80 characters) so you can identify it later — for example, `CI pipeline` or `Home automation`.
  </Step>
  <Step title="Copy and store your key">
    Copy the full key value immediately. It is displayed only once. Treat it like a password and store it in a secret manager or environment variable.
  </Step>
</Steps>

<Note>
  The full key is shown only once at creation time. If you lose it, you must revoke it and create a new one.
</Note>

## Key Limits by Plan

Your plan determines how many active keys you can hold and how many requests those keys can make per hour.

| Plan | Active Keys | Requests per Hour |
|---|---|---|
| Free | 1 | 60 |
| Premium | 5 | 1,000 |

## Using Your Key in Requests

Include your API key as a Bearer token in the `Authorization` header of every request.

<CodeGroup>
```bash cURL
curl -H "Authorization: Bearer subm_YOUR_KEY" \
  https://sub.jerrylu.app/api/v1/subscriptions
```

```python Python
import requests

headers = {"Authorization": "Bearer subm_YOUR_KEY"}
response = requests.get(
    "https://sub.jerrylu.app/api/v1/subscriptions",
    headers=headers,
)
print(response.json())
```

```javascript JavaScript
const response = await fetch(
  "https://sub.jerrylu.app/api/v1/subscriptions",
  {
    headers: {
      Authorization: "Bearer subm_YOUR_KEY",
    },
  }
);
const data = await response.json();
console.log(data);
```
</CodeGroup>

## Authentication Error Responses

When authentication fails, the API returns a JSON error body along with the relevant HTTP status code.

| Error Code | Meaning |
|---|---|
| `invalid_api_key` | The key does not exist, has been revoked, or is malformed |
| `rate_limit_exceeded` | You have sent too many requests; check `X-RateLimit-Reset` |

All error responses follow this structure:

```json
{
  "error": {
    "code": "invalid_api_key",
    "message": "Invalid API key"
  },
  "requestId": "a1b2c3d4-e5f6-7890-abcd-ef1234567890"
}
```

The `requestId` field is a unique identifier for the request. Include it when contacting support to help diagnose issues.

## Revoking a Key

To revoke an API key, open **Developer API** in the user menu and click **Revoke** next to the key you want to remove. Revocation takes effect immediately — any in-flight requests using that key will fail with `invalid_api_key`.

<Warning>
  Treat your API key like a password. Do not commit it to source code, include it in client-side bundles, or share it publicly. If a key is ever compromised, revoke it immediately from the Developer API menu and create a replacement.
</Warning>
